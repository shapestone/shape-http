## Reporting Bugs

Found a bug? Please use our **[bug report template](.github/ISSUE_TEMPLATE/bug_report.yml)**.

### Before Reporting

1. **Search existing issues** - Check if it's already reported
2. **Verify your version** - Make sure you're using the latest release
3. **Check the CHANGELOG** - It may be a known issue that's already fixed
4. **Create minimal reproduction** - Reduce to the smallest code that shows the problem

### What to Include

- **shape-http version** - Check with `go list -m github.com/shapestone/shape-http`
- **Go version** - Run `go version`
- **Operating system**
- **Minimal code sample** that reproduces the issue
- **Expected behavior**
- **Actual behavior**
- **Error messages** - Full error output if applicable

## Reporting Security Vulnerabilities

**Do NOT open public issues for security vulnerabilities.**

- **Preferred:** Use [GitHub Private Vulnerability Reporting](https://github.com/shapestone/shape-http/security/advisories/new)
- **Alternative:** Email security@shapestone.com

See our complete [Security Policy](SECURITY.md) for details.
